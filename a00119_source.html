1 Input Files
2 ===========
3 
4 The following input files are required to run HyPar. They are indicated as being mandatory or
5 optional. If optional, then default values are used if not found.
6 
7 \b Note: It is best to start with the provided examples, understand their input files based on the
8 information on this page, and then modify them to run other cases.
9 
10 \section solver_inp solver.inp
11 
12 Requirement: \b mandatory
13 
14 Read by: ReadInputs()
15 
16 Description: Specify the main simulation-related parameters.
17 
18 Format: ASCII text
19 
20  begin
21  <keyword> <value>
22  <keyword> <value>
23  <keyword> <value>
24  ...
25  <keyword> <value>
26  end
27 
28 where the list of keywords and their type are:\n
29 Keyword name | Type | Variable | Default value
30 ------------------ | ------------ | ----------------------------- | ------------------------
31 ndims | int | #HyPar::ndims | 1
32 nvars | int | #HyPar::nvars | 1
33 size | int[ndims] | #HyPar::dim_global | must be specified
34 iproc | int[ndims] | #MPIVariables::iproc | must be specified
35 ghost | int | #HyPar::ghosts | 1
36 n_iter | int | #HyPar::n_iter | 0
37 restart_iter | int | #HyPar::restart_iter | 0
38 time_scheme | char[] | #HyPar::time_scheme | euler
39 time_scheme_type | char[] | #HyPar::time_scheme_type | none
40 hyp_space_scheme | char[] | #HyPar::spatial_scheme_hyp | 1
41 hyp_flux_split | char[] | #HyPar::SplitHyperbolicFlux | no
42 hyp_interp_type | char[] | #HyPar::interp_type | characteristic
43 par_space_type | char[] | #HyPar::spatial_type_par | nonconservative-1stage
44 par_space_scheme | char[] | #HyPar::spatial_scheme_par | 2
45 dt | double | #HyPar::dt | 0.0
46 conservation_check | char[] | #HyPar::ConservationCheck | no
47 screen_op_iter | int | #HyPar::screen_op_iter | 1
48 file_op_iter | int | #HyPar::file_op_iter | 1000
49 op_file_format | char[] | #HyPar::op_file_format | text
50 ip_file_type | char[] | #HyPar::ip_file_type | ascii
51 input_mode | char[] | #HyPar::input_mode | serial
52 output_mode | char[] | #HyPar::output_mode | serial
53 op_overwrite | char[] | #HyPar::op_overwrite | no
54 model | char[] | #HyPar::model | must be specified
55 immersed_body | char[] | #HyPar::ib_filename | "none"
56 
57 \b Notes:
58 + "ndims" \b must be specified \b before "size" and "iproc".
59 + if "input_mode" or "output_mode" are set to "parallel" or "mpi-io",
60  the number of I/O ranks must be specified right after as an integer.
61  For example:
62 
63  begin
64  ...
65  input_mode parallel 4
66  ...
67  end
68 
69  This means that 4 MPI ranks will participate in file I/O (assuming
70  total MPI ranks is more than 4) (see ReadArrayParallel(),
71  WriteArrayParallel(), ReadArrayMPI_IO() ).
72  - The number of I/O ranks specified for "input_mode" and "output_mode"
73  \b must \b be \b same. Otherwise, the value for the one specified last
74  will be used.
75  - The number of I/O ranks must be such that the total number of MPI ranks
76  is an integer multiple. Otherwise, the code will use only 1 I/O rank.
77 + If any of the keywords are not present, the default value is used, except
78  the ones whose default values say "must be specified". Thus, keywords that
79  are not required for a particular simulation may be left out of the
80  solver.inp input file. For example,
81  - a #Euler1D simulation does not need "par_space_type" or "par_space_scheme"
82  because it does not have a parabolic term.
83  - unless a conservation check is required, "conservation_check" can be left
84  out and the code will not check for conservation.
85  - "immersed_body" need not be specified if there are no immersed bodies present.
86  \b NOTE: However, if it is specified, and a file of that filename does not
87  exist, it will result in an error.

89 \section boundary_inp boundary.inp
90 
91 Requirement: \b mandatory
92 
93 Read by: InitializeBoundaries()
94 
95 Description: Specify boundary conditions.
96 
97 Format: ASCII text
98 
99  nb
100  boundary_type spatial_dimension face [extent]
101  boundary_type spatial_dimension face [extent]
102  ...
103  boundary_type spatial_dimension face [extent]
104 
105 where
106 + \b nb is the number of boundaries
107 + This is followed by at least \b nb rows, each row specifying:
108  - \b boundary_type - Type of boundary, eg. #_PERIODIC_, #_EXTRAPOLATE_, #_DIRICHLET_, etc. See include/boundaryconditions.h for a full list.
109  - \b spatial_dimension - The spatial dimension along which this boundary condition applies. Must be in the range 0, 1, ..., #HyPar::ndims-1
110  - \b face - The face on which this the boundary acts (1 means the left face along the dimension, or the low index end, -1 means the right face
111  along the dimension, or the high index end).
112  - [\b extent] - The extent of this boundary (in terms of coordinates, not grid indices). Written as (2 X #HyPar::ndims) real numbers:
113  xmin_0 xmax_0 xmin_1 xmax_1 ... ... xmin_(ndims-1) xmax_(ndims-1), where [xmin_i, xmax_i] define the extent along
114  spatial dimension i. Note that the xmin_i and xmax_i for i = spatial_dimension are ignored, but should be there for the
115  sake of a uniform format (they can be set as 0).
116 + For the following boundary types, an additional line is required right below the line specifying that boundary with inputs specific to that
117  boundary type:
118  - #_DIRICHLET_: the next line should specify the boundary value of the solution at that boundary (the value is assumed to be constant in space
119  and time). Each component of the solution must be specified (i.e. #HyPar::nvars real numbers).
120 
121  u[0] u[1] ... u[#HyPar::nvars-1]
122 
123  - #_SLIP_WALL_, #_NOSLIP_WALL_, #_SW_SLIP_WALL_, or #_SW_NOSLIP_WALL_: the next line should specify the wall velocity (assumed to be constant
124  in space and time). Each component of the velocity must be specified (i.e. #HyPar::ndims real numbers).
125 
126  u[0] u[1] ... u[#HyPar::ndims-1]
127 
128  - #_SUBSONIC_INFLOW_: the next line must specify the inflow density and velocity vector (assumed to be constant in space and time). Each component
129  of the velocity must be specified (i.e. #HyPar::ndims+1 real numbers)
130 
131  rho u[0] u[1] ... u[#HyPar::ndims-1]
132 
133  - #_SUBSONIC_OUTFLOW_: the next line must specify the outflow pressure (assumed to be constant in space and time), i.e., one real number
134 
135  p
136 
137  - #_SUPERSONIC_INFLOW_: the next line must specify the inflow density, velocity, and pressure (assumed to be constant in space and time). Each component
138  of the velocity must be specified (i.e. #HyPar::ndims+2 real numbers)
139 
140  rho u[0] u[1] ... u[#HyPar::ndims-1] p
141 
142  Note that this boundary can also be implemented through the #_DIRICHLET_; however, the flow variables must be converted to the conserved variable
143  form to specify the Dirichlet boundary value:
144 
145  rho rho*u[0] rho*u[1] ... rho*u[#HyPar::ndims-1] E (internal energy)
146 
147  - #_TURBULENT_SUPERSONIC_INFLOW_: the next line must specify the inflow mean density, velocity, and pressure (assumed to be constant in space and time),
148  i.e, #HyPar::ndims+2 real numbersm and the name of the file that contains the unsteady fluctuation data.
149 
150  rho u[0] u[1] ... u[#HyPar::ndims-1] p filename

251 \section petscrc .petscrc
252 
253 Requirement: \b mandatory <B>if using PETSc time integration</B>; if absent, HyPar will use native time integration.
254  If compiled without PETSc, this file is not required.
255 
256 Read by: PETSc
257 
258 Description: This file contains the input flags required by PETSc time integration and its associated features.
259 
260 Format: ASCII text
261 
262 \b Note: The contents of this file may also be specified as command line flags.
263 
264 This file contains all the inputs required for the PETSc time integrators. See PETSc documentation (http://www.mcs.anl.gov/petsc/documentation/)
265 for all the inputs that PETSc needs, or <B>see the PETSc examples</B> for the inputs relevant to HyPar. In addition, following
266 are the HyPar-specific inputs (they are all optional, if not specified, default values are used):
267 + <B>-use-petscts</B>: If this flag is specified, PETSc time integration is used. If not specified, native time integration is used
268  (default).
269 + <B>-jfnk_epsilon <value></B>: specify \f$\epsilon\f$ parameter for the directional-derivative-based approximation of the Jacobian
270  (relevant only for implicit and IMEX time integration) (default: \f$10^{-6}\f$).
271 + <B>-with_pc</B>: If this flag is specified, a preconditioning matrix will be assembled for use with the preconditioners available
272  in PETSc. If not specified, no preconditioning will be used (default).
273 
274 In addition, if an IMEX time integrator (TSARKIMEX - http://www.mcs.anl.gov/petsc/petsc-current/docs/manualpages/TS/TSARKIMEX.html)
275 is being used, the following terms specify how the hyperbolic, parabolic,
276 and source terms are integrated in time (explicitly or implicitly):
277 + <B>-hyperbolic_explicit</B>: treat hyperbolic term explicitly (\b default).
278 + <B>-parabolic_explicit</B>: treat parabolic term explicitly.
279 + <B>-source_explicit</B>: treat the source term explicitly.
280 + <B>-hyperbolic_implicit</B>: treat hyperbolic term implicitly.
281 + <B>-parabolic_implicit</B>: treat parabolic term implicitly (\b default).
282 + <B>-source_implicit</B>: treat the source term implicitly (\b default).

284 \section immersed_body Immersed Body
285 
286 Requirement: \b mandatory <B>if using immersed boundaries</B> (the keyword \a immersed_body is specified in
287  \b solver.inp); if absent, HyPar will not use immersed boundaries.
288 
289 Read by: IBReadBodySTL(), called by the initialization function for immersed boundaries InitializeImmersedBoundaries().
290 
291 File name: The name of this file <B>must</B> be the same as the value of the keyword \a immersed_body in \b solver.inp.
292 
293 Format: ASCII STL (https://en.wikipedia.org/wiki/STL_%28file_format%29)
294 
295 Notes:
296 + The normal defined in the STL file must be the <B>"outward"</B> normal, i.e., pointing outside the body.
297 + The geometry must be a closed one.
298 + Some sample STL files are available in \b hypar/Examples/STLGeometries/
